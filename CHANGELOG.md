# Changelog

## Unreleased

- Made filesystem safe-root construction fallible, deterministic, and bounded before listener startup: one through 64 configured absolute non-symlink directory entries are normalized, sorted, deduplicated, opened component-by-component without following links, and retained as lifetime-pinned descriptors with device/inode identities. Every clone, descendant operation, and create/copy/write grant issue/consume path uses the same pinned-identity contract rather than reopening a pathname, so root or ancestor rename/replacement cannot redirect a running runtime and a grant issued against a replacement fails binding. Added configured-root-path/descriptor-identity redaction and deterministic invalid-root, clone-sharing, replacement-resistance, and grant-mismatch regressions.
- Closed command enablement structurally at the Rust crate boundary: the package binary compiles the server module graph in its own crate and alone can call the two crate-private command-capable router builders, while all twelve public library routers—including copy-file and all-filesystem constructors—hard-code command execution disabled. Ordinary dependency and selected-workspace-member compile probes first build the safe public API and then prove removed authority symbols, raw execution types, binary-private builders, and the former public command-flag signatures remain unreachable.
- Attested command initialization by opening the exact-name absolute `current_exe` candidate without following its final component, independently opening `/proc/self/exe`, and requiring executable regular-file metadata with identical device/inode identity before always spawning `/proc/self/exe`. The first safe root is held by a no-follow directory descriptor, root aliases are rejected by device/inode, and children use `/proc/self/fd/<fd>` with the descriptor alive through execution. The shared supervisor independently caps profiles at 5 seconds, 16 KiB stdout, and 4 KiB stderr, rejects maximum-plus-one configurations before spawn, and grows output capacity fallibly only as bytes arrive.
- Tightened native command evidence v2 to exactly 34 MCP requests and complete provenance, artifact, and environment validation. Its combined adversarial phase starts from `/`, replaces both the executable pathname and safe-root pathname after initialization, and proves the loaded executable inode and working-directory descriptor remain pinned; wrong-name behavior is recorded precisely as command-lane fail-closed.
- Hardened live `copy_file` as an independently default-disabled Class 2 capability. Mutation now requires static authentication, its dedicated runtime gate, explicit `dry_run:false`, and a 60-second single-use opaque HMAC grant bound to principal, canonical session, both anchored roots and normalized paths, exact single-link source identity/size/high-resolution ctime/SHA-256, absent destination, and fixed no-replace posture.
- Added exact-binary `--issue-copy-file-grant`, process-global equivalent-authority replay/clock/capacity state, all-family wire isolation, actual-id response preflight, cancellation-safe detached worker ownership, create/copy/write publication serialization, hidden mode-`0700` quarantine staging, fixed mode `0600`, atomic `NOREPLACE`, path/content-free results, private terminal audits, and release/device/emulated v9 evidence.

- Made the write-grant wire payload fixed-size and opaque: 65 bytes encoded as 130 lowercase hexadecimal characters carry only a random grant ID, the signed write-family byte, a keyed domain-separated operation binding, and issued/expiry timestamps. Principal, session, path, content digest, and replacement identity remain binding inputs and are never serialized.
